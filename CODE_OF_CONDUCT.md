# Jp2 Air Quality Card Code of Conduct

## Our Commitment

We want this project and its community to be welcoming, respectful, and constructive for everyone.

By participating in this repository, you agree to help create an environment that is professional, inclusive, and focused on collaboration.

## Expected Behavior

All contributors, users, and maintainers are expected to:

- Be respectful in discussions and reviews
- Give constructive feedback
- Be patient and helpful with others
- Assume good intent when possible
- Stay focused on improving the project
- Respect different levels of experience and viewpoints

## Unacceptable Behavior

The following behavior will not be tolerated:

- Harassment, discrimination, or hateful language
- Personal attacks, insults, or intimidation
- Trolling, provocation, or deliberately disruptive behavior
- Publishing private or personal information without permission
- Spam, repeated off-topic comments, or unnecessary hostility
- Any conduct that makes others feel unsafe or unwelcome

## Scope

This Code of Conduct applies to all interactions related to this project, including:

- GitHub Issues
- Pull Requests
- Discussions
- Code reviews
- Project-related community exchanges

## Maintainer Responsibilities

Project maintainers are responsible for setting the tone of the community and enforcing this Code of Conduct fairly and consistently.

Maintainers may take any action they consider appropriate in response to behavior that violates these guidelines, including:

- Editing or removing comments
- Closing or locking discussions
- Rejecting contributions
- Temporarily or permanently restricting participation

## Reporting Issues

If you experience or witness unacceptable behavior, please report it privately to the project maintainers through GitHub instead of opening a public argument.

When possible, include:

- A clear description of what happened
- Links to the relevant discussion or content
- Any other context that may help review the situation

All reports should be handled seriously, reviewed promptly, and treated with discretion.

## Enforcement

Violations may result in warnings, content removal, temporary restrictions, or permanent bans from the project community, depending on the severity and repetition of the behavior.

## Good Faith

This Code of Conduct is meant to support a healthy open-source environment. It is not intended to limit reasonable technical disagreement, honest critique, or thoughtful discussion.

## Changes to This Policy

Project maintainers may update this Code of Conduct when needed to keep the community safe, productive, and aligned with the goals of the project.

## Thank You

Thank you for helping make this project respectful, collaborative, and useful for everyone.
